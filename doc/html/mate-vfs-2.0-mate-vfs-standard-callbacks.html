MATE_VFS_MODULE_CALLBACK_FILL_AUTHENTICATION

#define MATE_VFS_MODULE_CALLBACK_FILL_AUTHENTICATION "fill-authentication"

A module callback name used together with mate_vfs_module_callback_invoke() to ask the keyring manager for login data. It is expected to return stored or cached login data, but may not query the user.

The login data consists of a username, a password and a domain, and is used to access a resource. If the data is not suitable for accessing the resource, the MateVFSModule typically issues a MATE_VFS_MODULE_CALLBACK_FULL_AUTHENTICATION module callback to query the user right after the authentication failed.

in arg :

 A MateVFSModuleCallbackFillAuthenticationIn * passed to the application, specifying the authentication request. The application will usually proxy this request to the keyring manager of the desktop environment, but it can also handle the request itself.

out arg :

 A MateVFSModuleCallbackFillAuthenticationOut * passing the cached keyring data (i.e. username/password etc.) back to the module. Usually, this is data that was stored using MATE_VFS_MODULE_CALLBACK_SAVE_AUTHENTICATION, and is either stored permanently on disk or cached.

MATE_VFS_MODULE_CALLBACK_FULL_AUTHENTICATION

#define MATE_VFS_MODULE_CALLBACK_FULL_AUTHENTICATION "full-authentication"

A module callback name used together with mate_vfs_module_callback_invoke() to ask the user for login data. This includes username and password, but also special login choices like anonymous login.

in arg :

 A MateVFSModuleCallbackFullAuthenticationIn * passed to the application, specifying the authentication request.

out arg :

 A MateVFSModuleCallbackFullAuthenticationOut * passing the user's provided authentication data (i.e. his username/password etc.) back to the module.

enum MateVFSModuleCallbackFullAuthenticationFlags

typedef enum {
	MATE_VFS_MODULE_CALLBACK_FULL_AUTHENTICATION_PREVIOUS_ATTEMPT_FAILED = 1<<0,
	MATE_VFS_MODULE_CALLBACK_FULL_AUTHENTICATION_NEED_PASSWORD = 1<<1,
	MATE_VFS_MODULE_CALLBACK_FULL_AUTHENTICATION_NEED_USERNAME = 1<<2,
	MATE_VFS_MODULE_CALLBACK_FULL_AUTHENTICATION_NEED_DOMAIN = 1<<3,
	MATE_VFS_MODULE_CALLBACK_FULL_AUTHENTICATION_SAVING_SUPPORTED = 1<<4,
	MATE_VFS_MODULE_CALLBACK_FULL_AUTHENTICATION_ANON_SUPPORTED = 1<<5
} MateVFSModuleCallbackFullAuthenticationFlags;

These flags will be passed to the MATE_VFS_MODULE_CALLBACK_FULL_AUTHENTICATION callback, as part of the MateVFSModuleCallbackFullAuthenticationIn structure. The output data will be stored in a MateVFSModuleCallbackFullAuthenticationOut structure.

MATE_VFS_MODULE_CALLBACK_FULL_AUTHENTICATION_PREVIOUS_ATTEMPT_FAILED

 This is not the first login attempt, i.e. this callback was already invoked but the provided login data was not suitable for a successful login.

MATE_VFS_MODULE_CALLBACK_FULL_AUTHENTICATION_NEED_PASSWORD

 The application should ask the user for a password and set the password field of MateVFSModuleCallbackFullAuthenticationOut.

MATE_VFS_MODULE_CALLBACK_FULL_AUTHENTICATION_NEED_USERNAME

 The application should ask the user for a username and set the username field of MateVFSModuleCallbackFullAuthenticationOut.

MATE_VFS_MODULE_CALLBACK_FULL_AUTHENTICATION_NEED_DOMAIN

 The application should ask the user for a domain and set the domain field of MateVFSModuleCallbackFullAuthenticationOut.

MATE_VFS_MODULE_CALLBACK_FULL_AUTHENTICATION_SAVING_SUPPORTED

 The application may ask the user whether he wants to save the password. If this flag is not present, or the user does not want to save the password, the application must set the save_password field of MateVFSModuleCallbackFullAuthenticationOut to FALSE and its keyring field to NULL. If the save_password field is TRUE, the module invoking the MATE_VFS_MODULE_CALLBACK_FULL_AUTHENTICATION callback is expected to invoke the MATE_VFS_MODULE_CALLBACK_SAVE_AUTHENTICATION callback if the login attempt was successful with the username, password, domain and keyring fields of MateVFSModuleCallbackFullAuthenticationOut.

MATE_VFS_MODULE_CALLBACK_FULL_AUTHENTICATION_ANON_SUPPORTED

 The application should offer the user a choice to login anonymously (used for example by the FTP module). If the user requests anonymous login, MATE_VFS_MODULE_CALLBACK_FULL_AUTHENTICATION_OUT_ANON_SELECTED must be set in the out_flags field of MateVFSModuleCallbackFullAuthenticationOut.

MATE_VFS_MODULE_CALLBACK_SAVE_AUTHENTICATION

#define MATE_VFS_MODULE_CALLBACK_SAVE_AUTHENTICATION "save-authentication"

A module callback name used together with mate_vfs_module_callback_invoke() to request permanent or temporary storage of login data. The storage is typically done using a keyring manager.

Called after a successfull authentication, to allow the client to e.g. store the password for future use. It may be queried again within the current session (temporary storage) or in future sessions (permanent storage) using MATE_VFS_MODULE_CALLBACK_FILL_AUTHENTICATION.

This is typically called after the user provided login data with MATE_VFS_MODULE_CALLBACK_FULL_AUTHENTICATION.

in arg :

 A MateVFSModuleCallbackSaveAuthenticationIn * specifying the login data.

out arg :

 A MateVFSModuleCallbackSaveAuthenticationOut * (unused).
