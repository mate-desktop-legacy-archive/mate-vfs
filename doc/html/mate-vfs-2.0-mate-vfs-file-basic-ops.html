Description

MateVFS file operations are, for the most part, patterned after their POSIX equivalents, with the systematic difference that they accept URIs rather than paths on the local filesystem. This makes them easy to learn as if you are already familiar with basic commands such as open(), seek(), write(), etc you will feel right at home with MateVFS after learning a little about URIs.

enum MateVFSOpenMode

typedef enum {
        MATE_VFS_OPEN_NONE = 0,
        MATE_VFS_OPEN_READ = 1 << 0,
        MATE_VFS_OPEN_WRITE = 1 << 1,
        MATE_VFS_OPEN_RANDOM = 1 << 2,
	MATE_VFS_OPEN_TRUNCATE = 1 << 3
} MateVFSOpenMode;

Mode in which files are opened. If MATE_VFS_OPEN_RANDOM is not used, the file will be have to be accessed sequentially.

MATE_VFS_OPEN_NONE

 No access.

MATE_VFS_OPEN_READ

 Read access.

MATE_VFS_OPEN_WRITE

 Write access.

MATE_VFS_OPEN_RANDOM

 Random access.

MATE_VFS_OPEN_TRUNCATE

 Truncate file before accessing it, i.e. delete its contents.
