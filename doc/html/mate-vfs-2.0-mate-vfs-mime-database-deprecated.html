enum MateVFSMimeActionType

typedef enum {
	MATE_VFS_MIME_ACTION_TYPE_NONE,
	MATE_VFS_MIME_ACTION_TYPE_APPLICATION,
	MATE_VFS_MIME_ACTION_TYPE_COMPONENT
} MateVFSMimeActionType;

Warning

MateVFSMimeActionType is deprecated and should not be used in newly-written code.

This is used to specify the type of a MateVFSMimeAction.

MATE_VFS_MIME_ACTION_TYPE_NONE

 neither an application nor a component.

MATE_VFS_MIME_ACTION_TYPE_APPLICATION

 an application.

MATE_VFS_MIME_ACTION_TYPE_COMPONENT

 a component.

mate_vfs_mime_action_launch ()

MateVFSResult      mate_vfs_mime_action_launch        (MateVFSMimeAction *action,
                                                         GList *uris);

Warning

mate_vfs_mime_action_launch is deprecated and should not be used in newly-written code. MIME actions are deprecated, use mate_vfs_mime_application_launch() instead.

Launches the given mime action with the given parameters. If the action is an application the command line parameters will be expanded as required by the application. The application will also be launched in a terminal if that is required. If the application only supports one argument per instance then multiple instances of the application will be launched.

If the default action is a component it will be launched with the component viewer application defined using the mateconf value: /desktop/mate/application/component_viewer/exec. The parameters s and c in the command line will be replaced with the list of parameters and the default component IID respectively.

action :

 the MateVFSMimeAction to launch.

uris :

 parameters for the MateVFSMimeAction.

Returns :

 MATE_VFS_OK if the action was launched, MATE_VFS_ERROR_BAD_PARAMETERS for an invalid action. MATE_VFS_ERROR_NOT_SUPPORTED if the uri protocol is not supported by the action. MATE_VFS_ERROR_PARSE if the action command can not be parsed. MATE_VFS_ERROR_LAUNCH if the action command can not be launched. MATE_VFS_ERROR_INTERNAL for other internal and MateConf errors.
